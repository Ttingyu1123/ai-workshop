三個名詞先搞懂

整個流程只會用到三個新名詞，先用醫院的架構理解一次，後面就不會迷路。

GitHub 就像醫院的「病歷室」— 一個存放所有資料、可以追蹤每次修改紀錄的地方。Repository（倉庫）就像一份「病歷夾」— 一個獨立的資料夾，裡面放某個病人（某個網站專案）的所有檔案。GitHub Pages 就像把某份衛教單張「公告在醫院大廳」— 把病歷夾裡的網頁檔案變成大家都能上網看到的公開頁面。
GitHub

全世界最大的程式碼託管平台，也可以拿來存放任何檔案。註冊帳號完全免費。

Repository

一個專案的資料夾，裡面放你的網頁檔案（HTML、圖片等）。一個帳號可以開很多個。

GitHub Pages

GitHub 內建的免費網站託管功能，把 repository 裡的網頁檔案直接變成一個公開網址。

註冊與建立 Repository

全程都在網頁介面完成，不需要安裝任何軟體。

Step 1 註冊帳號

前往 github.com，點右上角 Sign up，用 email 註冊即可，完全免費。

Step 2 New repository

登入後點右上角「+」→「New repository」，開始建立一個新的病歷夾。

Step 3 命名

Repository name 填一個英文名稱，例如 my-clinic-site。這個名字之後會出現在網址裡。

Step 4 設為 Public

選擇 Public（公開）— 免費方案的 GitHub Pages 只能從公開 repo 發布。注意：Public 代表整份原始檔案都公開，不只是網站畫面 — 所以絕不要放內部文件、病人資料、密碼或 API 金鑰進這個病歷夾。

Step 5 建立

勾選「Add a README file」，然後點綠色的「Create repository」按鈕，病歷夾就建好了。

你的第一個網頁

網站的入口檔案一定要叫做 index.html — GitHub Pages 會自動把它當成首頁顯示。整個過程都在 GitHub 網頁上完成，不用裝任何軟體。

  1. 進到剛建立好的 repository 頁面
  2. 點「Add file」→「Create new file」
  3. 檔名欄位輸入 index.html
  4. 把下面的範例貼進內容欄位
  5. 捲到最下面，點綠色的「Commit changes」按鈕存檔
index.html — 最小可用範例
<!DOCTYPE html> <html lang="zh-TW"> <head> <meta charset="UTF-8"> <title>我的第一個網站</title> </head> <body> <h1>哈囉，這是我的網站！</h1> <p>這一頁是用 GitHub Pages 免費架設的。</p> </body> </html>
檔名要完全一樣：一定要是小寫的 index.html，多一個空格或大小寫不同，GitHub Pages 都認不出來，會出現 404 找不到頁面。

開啟 GitHub Pages

檔案存好了，還差最後一步「公告在大廳」— 打開 GitHub Pages 功能。

Step 1 進入 Settings

在 repository 頁面上方點「Settings」分頁。

Step 2 找到 Pages

左側選單找到「Code and automation」區塊下的「Pages」，點進去。

Step 3 Source 選 Deploy from a branch

「Build and deployment」底下的 Source 下拉選單選擇「Deploy from a branch」。

Step 4 Branch 選 main

Branch 下拉選「main」，資料夾維持「/ (root)」，點「Save」。

Step 5 等 1-2 分鐘

重新整理 Pages 設定頁面，會出現一條綠色訊息和你的網址，格式是 https://帳號.github.io/repo名/。點開就是你的網站了。

本站的網址就是這樣來的：帳號是 Ttingyu1123，repository 叫 ai-workshop，組合起來就是 https://ttingyu1123.github.io/ai-workshop/。

讓 AI 幫你寫網站

看到上面的 HTML 標籤覺得頭暈也沒關係 — 完全不用學 HTML。把你想要的網站描述清楚，交給 AI Agent（見 Part 7）或 ChatGPT，直接生出完整的 index.html，複製貼上到 GitHub 就好。

請 AI 生成衛教網站 — Prompt 範例
請幫我寫一個單頁的衛教網站，用純 HTML + CSS（不要任何框架），輸出完整的 index.html 檔案。 主題：糖尿病患者的日常照護 風格：溫暖、清晰易讀，適合中高齡病人閱讀，字要夠大 章節： 1. 什麼是糖尿病（簡短說明） 2. 血糖監測的正確方法 3. 飲食注意事項（可以吃什麼、避免什麼） 4. 什麼情況要立刻回診 行動呼籲：頁尾放一句「有任何疑問請洽您的主治醫師」，並附上診所的預約電話（我會自己填） 醫療內容限制：只根據我上面提供的重點撰寫，不可自行新增診療建議、藥物劑量或診斷標準；不確定的地方標示【需人工確認】。 請直接給我完整可用的程式碼，不要省略任何部分。
拿到 AI 生成的程式碼後，整段複製，貼到 GitHub 的「Add file → Create new file」→ 檔名 index.html，Commit 之後就上線了。想改內容，回去問 AI「幫我把第三段改成 XXX」，拿到新版程式碼再貼一次覆蓋即可。

常見問題

打開網址是 404 找不到頁面？

先等 1-2 分鐘再重新整理，剛開啟 Pages 需要一點部署時間。接著確認檔名真的是小寫的 index.html，最後回 Settings → Pages 確認狀態是「Your site is live」。

改了內容，網站卻沒更新？

通常是瀏覽器快取問題，按 Ctrl+F5（Mac 是 Cmd+Shift+R）強制重新整理。也可以確認 GitHub 上的檔案確實已經 Commit 成功。

圖片放上去了但不顯示？

檢查圖片檔名和程式碼裡引用的路徑是否完全一致，包含大小寫 —Photo.jpg 和 photo.jpg 在 GitHub Pages 上會被視為不同檔案。

⚠️ 網站是完全公開的

GitHub Pages 上的任何內容，全世界都能搜尋和看到。衛教內容絕對不可以含有病人姓名、病歷號、床號、診斷細節等任何可識別病人身分的資訊，即使只是當作範例截圖也不行。
