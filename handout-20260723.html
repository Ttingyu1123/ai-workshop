課堂示範　衛教文案＋配圖

這兩段是課堂示範用的 Prompt — 老師投影操作時，你可以在手機上跟著做，或回家再試一次。

示範 1　三分鐘生出衛教文案

打開 ChatGPT 或 Gemini，貼上送出：

Prompt 1 — 衛教文案（流感疫苗）
你是感染科的衛教護理師。請為65歲以上的長者，寫一份「流感疫苗接種」衛教單張文案：語氣親切易懂、避免專業術語，150字以內，最後條列3個接種前的注意事項
這段 Prompt 的公式：角色＋任務＋條件＋格式

角色（衛教護理師）＋任務（寫衛教單張文案）＋條件（65 歲以上長者、親切易懂）＋格式（150 字內、條列 3 點）— 四個都寫到，品質立刻不一樣；少一個，AI 就自己猜一個。初稿不滿意就「追問」：「語氣再溫暖一點」「縮短成 100 字」「改成適合 LINE 群組轉發的版本」。

示範 2　為衛教文案配一張插圖

在同一個對話接著輸入：

Prompt 2 — 衛教插圖（接種情境）
請畫一張溫暖手繪風格的插圖：一位長者在診所接種流感疫苗，護理師在旁微笑陪伴，柔和的色調，適合放在衛教單張上

注意它記得上下文 — 插圖與剛才的文案是同一個主題，不需要重新描述一次。這種「接著上一句往下做」的工作方式，是 AI 對話與搜尋引擎最大的差別。

實作 ①　含標題文字的衛教海報

第一張圖，我們讓 AI 從零生出一張帶有中文標題的衛教海報 — 從打字到完成大約兩分鐘。

操作步驟

  1. 按下方 Prompt 3 的「📋 複製」。
  2. 打開 ChatGPT 或 Gemini（手機 App 或網頁版皆可）。
  3. 在對話框貼上 Prompt 3。
  4. 把「主題」「大標題」「副標」改成你想要的內容 — 第一次也可以先照原文生成，熟悉流程後再改。
  5. 左下角選擇「建立圖像」，或按「＋」選擇「創作圖像」，送出。
  6. 等待生成（約 1–2 分鐘）。
Prompt 3 — 衛教海報
生成一張衛教海報。 主題：預防流感 圖片上的文字： - 大標題：「預防流感 從手開始」（置頂，大字，白色） - 副標：「正確洗手 5 步驟」（標題下方，較小） 風格：扁平設計，主色調淡藍色，明亮溫暖 構圖：直式 3:4，標題在上方 1/3，下方 2/3 是洗手步驟插圖 注意：繁體中文字要清晰可讀，字體風格要統一
改字的訣竅

把主題換成你科別的內容試試：「大腸鏡檢查前準備」「正確量血壓」「疫苗接種須知」。注意這張海報刻意只放十幾個字 — AI 生圖的中文字數愈多愈容易出錯，字少是它能成功的關鍵。生出來若有錯字，同一段 Prompt 再生一次即可，每次結果都不同。

觀念插播　AI 是怎麼「畫」出圖的？

醫學類比

課堂上說過，文字 AI 像一位「憑經驗接話」的總醫師。生圖 AI 是它的親戚：它從一片雜訊出發，依照你的描述一步步「猜」出整張圖 — 像顯影中的 X 光片逐漸清晰。因為每次都是重新猜，同一段描述每次生成的結果都不同；也因為它是「畫出印象」而不是「排版文字」，中文字對它來說只是形狀相似的圖案，筆畫常常出錯。

由此推出生圖的三個實用技巧，等一下實作 ② 就會用到：

🔤 圖裡的中文字，愈少愈好

字多必錯。正式的衛教單張建議「圖歸圖、字歸字」：讓 AI 生成留白的圖，文字之後用 Canva 加上去最穩。

風格詞很好用

「扁平設計」「水彩」「3D 公仔風」— 換一個詞，整張圖的氣質就完全不同。實作 ② 的長 Prompt 有一大半都在描述風格。

🔁 多生幾張，挑最好的

每次結果都不同，不必一次就滿意。同一段 Prompt 生三、四張再挑選，是專業使用者的常態做法。

實作 ②　你的專屬袖珍公仔海報

這一張是今天的主角：一張「忙碌又荒謬的醫療人生」袖珍公仔海報，主角就是你的職業。這次我們多學一招 — 先請 AI 幫你改寫 Prompt，再拿改好的 Prompt 去生圖。

操作步驟

  1. 按下方 Prompt 4 的「📋 複製」，貼到對話框。
  2. 修改第一行：把「職類」改成你的職業（護理師、藥師、醫檢師、放射師、營養師…），「標題」改成你想要的標題，按「傳送」。
  3. AI 會輸出一段改寫完成的完整提示詞 — 按訊息上的「複製」。
  4. 另外開啟一個「新對話」，貼上剛剛複製的提示詞。
  5. （可不做）按左下角「＋」選擇「新增照片和檔案」，上傳自己的照片 — 公仔會長得像你。不上傳也能生成，AI 會自行設計人物。
  6. 左下角選擇「建立圖像」，或按「＋」選擇「創作圖像」，送出並等待生成。

上傳照片前，停 3 秒

只上傳自己的生活照。病人照片、含病歷資訊的工作截圖、有同事入鏡且未經同意的照片 — 一律不上傳。免費版 AI 工具上傳的內容可能被用於模型訓練，送出前請當成「會被別人看到」來判斷。

為什麼要先「改寫」再生圖？

這段 Prompt 裡的桌面道具、背景場景、八個小分身動作，都是針對「感染科醫師」設計的。與其自己逐項改，不如把改寫工作也交給 AI — 它會把培養皿換成藥櫃（藥師）或檢體架（醫檢師），連小分身的動作都一起換成你職業的日常。這就是課堂教的「對話式修改」：AI 的產出，可以再交給 AI 加工。改寫結果裡的小分身動作若不合你意，直接告訴它「第 3 個動作改成…」再要一版即可。

安全必修　三條紅線

工具再方便，這三條底線不變 — 它們是今天這堂課最重要的八分鐘：

1️⃣ 病人資料，絕不輸入

姓名、病歷號、出生日期、可辨識的照片 — 任何 AI 工具都不輸入。免費工具的輸入內容可能被用於模型訓練。

2️⃣ AI 產出，查證才用

AI 的目標是把話接得「順」，不是接得「對」。文獻引用、藥物劑量、精確數字，發出前必查證。

3️⃣ 對外文件，人工把關

要掛你名字發出去的東西，最終版本由你負責。AI 是很會寫的助理，不是權威。

想問臨床問題不是不行，而是先「去識別化」— 把「這個病人」變成「一種情境」再問：

不行

「病人王○○，病歷號 1234567，肺炎住院中……幫我寫病摘」

可以

「一位 70 歲糖尿病患者因社區型肺炎住院，請列出出院衛教重點」

需要給 AI 看截圖時，先用網站的「圖片去識別化」小工具框選塗黑 — 全程在自己裝置上處理、不上傳。完整內容見 Part 3 — AI 安全與隱私（本站唯一的必修頁）。

實作 ③　讓海報動起來

把剛剛生成的公仔海報變成一支「活起來的袖珍模型」短片：主角眨眼、咖啡冒煙、小分身各忙各的 — 而畫面上的文字一個都不能動。

操作步驟

  1. 按下方 Prompt 5 的「📋 複製」。
  2. 打開 Gemini App。
  3. 左下角按「＋」選擇「建立影片」。
  4. 上傳實作 ② 生成的公仔海報（這段 Prompt 的第一句就要求以上傳的圖作為第一幀）。
  5. 右下角選擇「直向（9:16）」。
  6. 貼上 Prompt 5，送出並等待生成 — 影片比圖片久，通常需要數分鐘。

觀念插播　為什麼這段 Prompt 一半都在「禁止」？

AI 影片最常失敗的原因不是「動不起來」，而是「動太多」：鏡頭亂飄、人物走位、文字融化變形。所以這段 Prompt 的策略是「鎖鏡頭、鎖文字、只做微動作」— 眨眼、呼吸、咖啡冒煙，動得少反而像。這是圖生影片成功率最高的入門姿勢：先有一張好圖，再給它一點點生命。

額度與替代方案

影片生成消耗的運算量遠大於圖片，免費帳號有每日次數限制（以 App 顯示為準），生成一支約 8 秒。額度用完或想在電腦上操作，同一段 Prompt 也能貼到 Google Flow（免費層每日 50 credits），完整教學見 Part 6 — Google Flow 實戰；影片工具全覽見 Part 5 — AI 影片製作。

觀念插播　這就是「Agent」

遊戲標題 - 簡短背景故事 - 「開始遊戲」按鈕 - 「遊戲說明」按鈕 - 音效開關 - 清楚的醫院夜間視覺氛圍 ## 第一關：護理站 場景元素可包括： - 交班白板 - 病房配置圖 - 護理紀錄 - 電話 - 抽屜 - 時鐘 - 閃爍的緊急照明 謎題方向： 玩家必須比較交班紀錄、床號與時間資訊，找出正確病房編號。 完成後取得： - 一張檢驗申請單 - 或一組可用於檢驗室的識別碼 謎題必須依靠觀察與推理，不能只靠隨機猜測。 ## 第二關：檢驗室 場景元素可包括： - 檢體架 - 血液分析儀 - 條碼標籤 - 檢驗報告 - 冰箱 - 顯微鏡 - 警示燈 謎題方向： 玩家根據上一關取得的識別碼，找到對應檢體，再依提示排列數值、色碼、符號或檢驗項目。 完成後取得： - 藥品代碼 - 藥櫃位置 - 或一張部分損毀的處方 不得要求玩家具備專業醫學知識；所有必要資訊都必須在遊戲中提供。 ## 第三關：藥局 場景元素可包括： - 藥櫃 - 處方籤 - 分裝台 - 藥袋 - 冰箱 - 條碼掃描器 - 上鎖抽屜 謎題方向： 玩家根據檢驗室獲得的線索，從多個藥袋、圖示、編號或顏色中找出正確組合。 完成後取得： - 機房門禁卡 - 維修代碼 - 或四位數密碼的一部分 藥名可以使用虛構名稱，避免形成真實用藥建議。 ## 第四關：地下機房 場景元素可包括： - 伺服器機櫃 - 電力控制盤 - 網路拓撲圖 - 門禁終端 - 故障螢幕 - 備援電源 - 資料恢復主機 謎題方向： 玩家必須整合前三關獲得的資訊，完成最後的電力、網路或資料排序謎題。 最後輸入唯一正確的恢復密碼，找回病例並解除醫院封鎖。 # 結局 完成最終謎題後，顯示： - 病例恢復成功 - 醫院照明逐步恢復 - 玩家成功離開 - 遊戲完成時間 - 使用提示次數 - 找到的隱藏物件數量 - 「重新遊玩」按鈕 可加入一個合理的小反轉，但不能破壞故事邏輯。 # 謎題設計規則 四個謎題必須符合以下條件： 1. 每個謎題都有唯一解。 2. 所有必要線索都能在遊戲中找到。 3. 前一關的結果會影響下一關。 4. 不可依靠外部搜尋或專業知識。 5. 不可使用純粹隨機答案。 6. 不可因操作順序不同而導致無法破關。 7. 玩家輸入錯誤時，要提供合理回饋。 8. 玩家不得因誤操作永久失去必要物品。 9. 提示內容應逐步揭露，不可第一次提示就直接給答案。 10. 請自行驗證所有謎題從線索到答案的邏輯完整性。 # 核心功能 遊戲至少包含： - 四個完整場景 - 場景切換 - 可點擊互動物件 - 物品欄 - 物品選取與使用 - 場景調查文字 - 對話框或資訊面板 - 謎題輸入介面 - 錯誤答案回饋 - 三階段提示系統 - 遊戲進度顯示 - 音效開關 - 重新開始功能 - 勝利畫面 - 遊戲計時 - 提示使用次數統計 - localStorage 自動保存進度 - 清除存檔並重新開始 - 鍵盤與滑鼠基本操作 - 行動裝置觸控操作 - 基本無障礙標籤 # 視覺風格 整體採用： - 電影感深夜醫院 - 懸疑但不恐怖 - 深藍、青綠、冷白與少量警示紅 - 柔和光影 - 清楚易讀的繁體中文 - 現代化遊戲 UI - 醫療資訊系統與復古解謎介面的混合風格 - 桌面版與手機版都要有良好可讀性 避免： - 血腥畫面 - 驚嚇圖像 - 真實病人照片 - 過度陰暗而看不清互動物件 - 大量小字 - 低對比文字 - 廉價預設按鈕 - 只有文字與色塊的簡陋介面 # 畫面素材規則 請自行製作遊戲所需的視覺素材。 優先順序： 1. 自行繪製 SVG 2. CSS 圖形與光影 3. Canvas 繪圖 4. 若系統具備圖像生成工具，可自行生成場景背景並整合進遊戲 所有圖像必須保持統一風格。 如果無法直接呼叫生圖工具，仍須使用 SVG、CSS、圖示與版面設計完成具有完整視覺層次的遊戲，不可因此省略畫面。 不要使用來源不明的外部圖片。 # 音效 可加入簡單音效，例如： - 按鈕點擊 - 解鎖 - 取得物品 - 輸入錯誤 - 電力恢復 - 伺服器啟動 音效必須可以關閉。 若無法合法取得音效，可使用 Web Audio API 生成簡單提示音，不要因缺乏音效素材而停止開發。 # 介面配置 建議包含： - 上方：場景名稱、進度、計時器 - 中央：主要場景 - 側邊或底部：物品欄 - 角落：提示、音效、重新開始 - 彈出式面板：文件、謎題與調查內容 互動物件應有適度的 hover、focus 或觸控提示，但不要讓答案過度明顯。 # 狀態管理 請建立清楚的遊戲狀態，包括： - 目前場景 - 已調查物件 - 已取得物品 - 已完成謎題 - 已解鎖區域 - 玩家選取中的物品 - 提示使用階段 - 遊戲開始時間 - 已經過時間 - 音效設定 - 隱藏物件紀錄 - 遊戲是否完成 重新載入頁面後，遊戲應能從最近進度繼續。 # 測試要求 完成實作後，請實際測試並修正問題。 至少測試： 1. 從新遊戲開始到正常破關的完整流程。 2. 每一關輸入正確答案。 3. 每一關輸入錯誤答案。 4. 未取得必要物品時嘗試進入下一關。 5. 對同一物件重複互動。 6. 不按預期順序調查物件。 7. 使用提示系統。 8. 重新整理頁面後讀取存檔。 9. 清除存檔並重新開始。 10. 勝利後重新遊玩。 11. 手機尺寸畫面。 12. 桌面尺寸畫面。 13. 鍵盤焦點與基本無障礙操作。 14. 所有按鈕是否有效。 15. 是否存在無法繼續、無限循環或死局。 16. 四個謎題是否真的具有唯一解。 17. 所有線索文字與最終答案是否一致。 發現問題後直接修正並重新測試，不要只列出尚未解決的錯誤。 # 程式品質 請確保： - 結構清楚 - 函式與變數命名合理 - 避免重複程式碼 - 不在 console 留下錯誤 - 不建立無用途的套件依賴 - 不把主要內容全部塞在單一巨大函式 - 重要邏輯有必要的註解 - 介面文字使用繁體中文 - 遊戲資料與畫面邏輯適度分離 # 最終交付要求 最終交付必須包含： 1. 可直接執行的完整遊戲檔案。 2. 清楚的啟動方式。 3. 遊戲操作說明。 4. 完整破關流程的測試結果。 5. 已完成與已驗證功能清單。 6. 如有任何技術限制，必須如實說明。 7. 不要只提供部分程式碼。 8. 不要只提供設計稿。 9. 不要只提供開發計畫。 10. 不要宣稱已測試實際上未執行的功能。 請以「可完整遊玩、可驗證、無假功能」為最高優先級，視覺華麗程度其次。
